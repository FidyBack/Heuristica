Relatório Intermediário - Abel Cavalcante¶

Este relatrio apresenta o resultado de trs projetos, todos calculando o alinhamento de duas sequncias de DNA distintas:

  • A primeira usa a Heurstica de Alinhamento Local de Smith-Waterman;
  • A segunda usa um algoritmo Local para Alinhamento;
  • E, por fim, usado uma tcnica de Busca Exaustiva para Alinhamento de Sequncias.

Heurística de Alinhamento Local de Smith-Waterman¶

Nesse primeiro mtodo, foi utilizado a Heurstica de Alinhamento Local de Smith-Waterman. Para us-la e test-la, uma das sequncia foi mantida com tamanho 50 e a outra varia entre 10 e 1.000.000:

In [ ]:
import subprocess, os
import matplotlib.pyplot as plt
In [ ]:
means1 = []
files = 6

for i in range(files):
    times = []
    for filename in os.listdir(f'entradas_{i+1}'):
        with open(f'entradas_{i+1}/{filename}', 'r') as f:
            proc = subprocess.run(['./her'], stdin=f, stdout=subprocess.PIPE)
            out = proc.stdout.decode('utf-8')

            if 'Tempo' in out:
                times.append(float(out.split('\n')[-2].split(' ')[-2]))

    means1.append(sum(times) / len(times))

Cada tamanho foi testado 10 vezes e foi feita uma mdia dos tempos obtidos. Assim, foi feito um grfico juntando o tempo mdio para cada tamanho de entrada, em ms:

In [ ]:
sizes1 = ['10', '100', '1000', '10000', '100000', '1000000']

plt.figure(figsize = (10, 7))
plt.plot(sizes1, means1)
plt.xlabel('Tamanho do arquivo')
plt.ylabel('Tempo de execuo (ms)')
plt.title('Tempo de execuo por tamanho do arquivo')
plt.show()

importante salientar que, para tamanhos na magnitude de 10.000.000, o programa comea a falhar:

Ao explorar mais a fundo, possvel observar que enviado um 'Killed' para o programa, o que pode-se supor que o poder computacional no suficiente para comportar o tamanho da entrada ou o cdigo no otimizado o suficiente paar essas entradas.

g++ -Wall -O3 -g heuristica.cc -o her

callgrind_annotate callgrind.out.63728 heuristica.cc > analise_calgrind_heuristica
